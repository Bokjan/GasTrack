# GasTrack 系统架构设计

## 1. 整体架构

采用前后端分离的 B/S 架构，后续扩展小程序端。

```

│  │(React)   │  │ (Taro/原生)   │  │  (React)      │  │
│  └────┬─────┘  └──────┬───────┘  └───────┬───────┘  │
```

## 2. 架构决策

### 2.1 为什么选择 Monolith 而非微服务？

**第一阶段采用模块化单体（Modular Monolith）：**
- 团队规模小，微服务运维成本高
- 业务复杂度适中，单体足以应对
- Go 的 package 机制天然支持模块化，后续可拆分
- 当用户量超过 10 万时再考虑拆分

### 2.2 前后端通信

- **RESTful API** 为主（CRUD 操作直观）
- 后续可引入 **GraphQL** 优化移动端数据获取 (P2)
- API 版本控制：URL Path 方式 `/api/v1/`

### 2.3 认证方案

```
┌──────┐    ①登录请求     ┌──────┐
│Client├──────────────────►│Server│
│      │◄──────────────────┤      │
└──┬───┘ ②Access+Refresh  └──┬───┘
   │      Token               │
   │  ③携带 Access Token      │
   │  请求业务 API             │
   ├──────────────────────────►│
   │                           │
   │  ④Token 过期              │
   │  用 Refresh Token 续期    │
   ├──────────────────────────►│
   │◄──────────────────────────┤
   │  ⑤返回新 Access Token     │
```

- Access Token：JWT，15 分钟有效
- Refresh Token：存数据库，7 天有效，支持吊销

## 3. 模块划分

| 模块 | 职责 | 依赖 |
|------|------|------|
| AuthModule | 登录注册、JWT | UserModule |
| UserModule | 用户 CRUD、偏好设置 | - |
| VehicleModule | 车辆 CRUD | UserModule |
| FuelRecordModule | 加油记录 CRUD | VehicleModule |
| StatsModule | 统计计算、报表生成 | FuelRecordModule |
| GroupModule | 家庭群组管理 | UserModule, VehicleModule |
| I18nModule | 多语言资源管理 | - |
| UploadModule | 文件上传（头像/车辆照片） | - |
| NotificationModule | 通知推送 | UserModule |
| AdminModule | 后台管理 | 全部模块 |

## 4. 多语言方案

### 4.1 前端 i18n
- 使用 `react-i18next` 库
- 翻译文件按语言分目录：`locales/zh-CN/`, `locales/en-US/`
- 按模块拆分翻译文件（懒加载）
- 语言检测优先级：用户设置 > URL 参数 > 浏览器语言 > 默认(en)

### 4.2 后端 i18n
- 使用 `go-i18n` 库
- API 错误消息国际化
- 通过 `Accept-Language` Header 传递语言偏好

## 5. 多币种与单位方案

### 5.1 存储策略
- 数据库统一用**原始值**存储（用户输入什么存什么）
- 同时记录币种代码和单位制
- **不做自动汇率转换**，只做展示层的单位换算

### 5.2 单位换算引擎
```
用户输入 → 原始值 + 原始单位存 DB
         → 展示时按用户偏好换算
         → 统计时按统一单位计算后再转换
```

支持的换算：

**燃油车**：
| 维度 | 公制(欧标) | 公制(日标) | 英制 |
|------|-----------|-----------|------|
| 容量 | 升 (L) | 升 (L) | 加仑 (gal) |
| 距离 | 公里 (km) | 公里 (km) | 英里 (mi) |
| 油耗 | L/100km | km/L | MPG |

**电动车**：
| 维度 | 公制(欧标) | 公制(日标) | 英制 |
|------|-----------|-----------|------|
| 容量 | kWh | kWh | kWh |
| 距离 | 公里 (km) | 公里 (km) | 英里 (mi) |
| 电耗 | kWh/100km | km/kWh | mi/kWh |

注：km/L 是日本最常用的油耗表示方式，数值越大越省油（与 L/100km 相反）。
换算关系：km/L = 100 / (L/100km)，MPG = km/L × 2.352
